Programmable Virtual Camera Access Method library
Host Post-Processing

Additionally to the camera native Post-Processing, PVCAM supports a set of post-processing features that are executed on the host. Contrary to the camera post-processing, the host processing features are available for all cameras and are accessible using the regular PVCAM parameters, no dynamic discovery is needed.

Since the features are executed on the host, the host CPU and eventually GPU may be affected.

List of PVCAM's host post-processing features

  • PARAM_HOST_FRAME_ROTATE and PARAM_HOST_FRAME_FLIP - these parameters allow virtual sensor rotation. This feature is useful in situation where the camera cannot be physically rotated. The virtual sensor rotation and flipping can be also enabled by environment variables, this allows existing applications to use this feature without any changes to the application. Please refer to the parameter documentation for more details.
  • PARAM_HOST_FRAME_SUMMING_ENABLED and related parameters - these parameters control PVCAM's internal frame-summing module. This feature can be used to increase the camera's dynamic range. This feature allows 16-bit and 32-bit output format. Please note that enabling this feature may change the frame output format. Applications that support these parameters should also use the related PARAM_BIT_DEPTH_HOST and PARAM_IMAGE_FORMAT_HOST parameters that report the host-side frame format.
  • PARAM_HOST_FRAME_DECOMPRESSION_ENABLED - enables or disables the PVCAM's internal decompression module. This is an advanced parameter and should be used only in very specific cases. For example, some camera models use bit-packing feature to reduce the frame size when transferring frames over limited-bandwidth interfaces. If this parameter is set to disabled state, PVCAM will not automatically decompress the incoming frames. This may allow the application to stream the compressed data directly to disk or RAM, however, the application will need to provide the decompression algorithm. See also PARAM_IMAGE_COMPRESSION_HOST and PARAM_IMAGE_COMPRESSION.
